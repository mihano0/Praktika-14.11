### Генетический алгоритм для оптимизации
Задача: реализовать простой генетический алгоритм для максимизации функции.
Требования:
- Использовать кодирование двоичными строками
- Размер популяции: 20
- Количество поколений: 50
- Вероятность кроссовера: 0.8
- Вероятность мутации: 0.1
- Вывести лучшее решение и его оценку функции
- 
### Объяснение работы алгоритма:


1. **Создание начальной популяции** — формируем набор случайных решений («особей»).
2. **Оценка приспособленности** — для каждой особи вычисляем «качество» решения с помощью функции `fitness()` (целевой функции).
3. **Селекция** — отбираем наиболее приспособленные особи для «размножения» (вероятность зависит от оценки).
4. **Кроссовер (скрещивание)** — комбинируем части решений двух сильных особей, создавая новых потомков (`crossover(p1, p2)`).
5. **Мутация** — случайно изменяем некоторые параметры у особей (`mutate(p1)`, `mutate(p2)`), чтобы избежать «застревания» в локальном оптимуме.
6. **Формирование нового поколения** — заменяем старую популяцию на новую, состоящую из потомков.
7. **Проверка сходимости** — анализируем, достигли ли оптимального решения:
   - заданное число поколений (`GEN_COUNT`) исчерпано;
   - популяция практически не меняется (особи схожи).
8. **Вывод результата** — находим особь с максимальной оценкой (`best.score`) и выводим её параметры (`best.value`) как итоговое решение.

**Временная сложность генетического алгоритма (ГА):** O(G × N × (F + S + C + M)), где:
- **G** — число поколений;
- **N** — размер популяции;
- **F** — время расчёта функции приспособленности (fitness) для 1 особи;
- **S** — время селекции (отбора) для 1 особи;
- **C** — время кроссовера (скрещивания) для 1 пары;
- **M** — время мутации для 1 особи.

**Коротко о факторах:**
- чем больше **N** и **G**, тем дольше работает ГА, но выше шанс найти хорошее решение;
- **длина хромосомы** влияет на C и M;
- сложность **F** может стать «бутылочным горлышком» (если оценка решения трудоёмка);
- итоговая сложность — полиномиальная, но может быть высокой для сложных задач.

### Контрольный вопрос 11. 
### Объясните принцип работы метаэвристики "Имитация отжига" и роль параметра температуры.
Принцип работы «Имитации отжига»:

Имитация отжига (АИО) — метаэвристический алгоритм оптимизации, вдохновлённый процессом отжига в металлургии. Его суть: постепенное улучшение решения за счёт случайных изменений с учётом «температуры» и качества новых вариантов.

Ключевые этапы:

- Рандомизация — алгоритм вносит случайные изменения в текущее решение, чтобы избежать локальных оптимумов.
- Оценка функции стоимости — определяется, насколько хорошо решение решает задачу (чем меньше значение, тем лучше).
- Охлаждение — постепенное снижение «температуры» (параметр алгоритма), что уменьшает вероятность принятия худших решений.
- Процесс останова — алгоритм завершается при достижении заданного числа итераций или требуемой точности.
  
Роль параметра «температура»:

Температура (T) в АИО — мера «разброса» принимаемых решений:
- Высокая температура → большой разброс решений, включая худшие варианты (способствует исследованию пространства решений).
- Низкая температура → меньший разброс, фокус на оптимальных решениях (способствует уточнению результата).
